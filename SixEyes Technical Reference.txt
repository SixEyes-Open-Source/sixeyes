------------------------------------------------------------
1. Project Purpose & Scope
------------------------------------------------------------

This project develops a low-cost, high-performance, 3D-printable
robotic arm for Vision–Language–Action (VLA) research.

The system is explicitly designed to:
- Enable data collection for VLA training
  (vision + joint state + action labels)
- Enable real-time VLA inference using an external compute device (laptop)
- Maximize hardware reuse and sustainability,
  especially reused 3D-printer components

The robotic arm is NOT an autonomous embedded-AI system.

Instead, it is a distributed embodied AI platform:
- ESP32-S3 handles real-time sensing, actuation, and safety
- Laptop handles perception, learning, and inference


------------------------------------------------------------
2. High-Level System Architecture
------------------------------------------------------------

2.1 Distributed Control Philosophy

Layer: Low-level control
Device: ESP32-S3
Responsibility: Motor control, sensing, safety, timing

Layer: Mid-level coordination
Device: USB-CDC protocol
Responsibility: Deterministic data & command transfer

Layer: High-level intelligence
Device: Laptop
Responsibility: VLA inference, dataset logging, planning

Layer: System orchestration
Device: ROS2 (Laptop)
Responsibility: Message passing, synchronization, tooling

3. Hardware Architecture (Revised)
------------------------------------------------------------

3.1 Leader Arm (Teleoperation Controller)

MCU:
- ESP32-S3-WROOM-1
- ADC1 used exclusively for potentiometers (clean analog domain)

Inputs:
- 6× Potentiometers
  (Base, Shoulder, Elbow, Wrist)

Leader Pinout (ADC1 only):

Pot 1  -> GPIO1
Pot 2  -> GPIO2
Pot 3  -> GPIO3
Pot 4  -> GPIO4
Pot 5  -> GPIO5
Pot 6  -> GPIO6


UART to Follower (cross-connected):

TX -> GPIO17
RX -> GPIO18


Power:
- 3.3V supplied from follower PCB
  OR optional local LDO if standalone


------------------------------------------------------------
3.2 Follower Arm (Actuated Arm)
------------------------------------------------------------

3.2.1 Degrees of Freedom (Finalized)

Base          : 1× NEMA17
Shoulder      : 2× NEMA17 (torque split)
Elbow         : 1× NEMA17
Wrist Pitch   : MG996R
Wrist Yaw     : MG996R
Gripper       : MG996R

Total:
- 4× Stepper channels
- 3× Servo channels
- 6 DOF


------------------------------------------------------------
3.3 Motor Drivers
------------------------------------------------------------

Drivers:
- 4× TMC2209
- PDN_UART mode
- StallGuard homing
- VIO = 3.3V
- VM  = 24V


Follower ESP32-S3-WROOM-1 Pinout
(TMC2209 ×4)

J1 (Base):
- STEP -> GPIO4
- DIR  -> GPIO5
- EN   -> GPIO6
- PDN_UART -> GPIO7

J2 (Shoulder A):
- STEP -> GPIO8
- DIR  -> GPIO9
- EN   -> GPIO10
- PDN_UART -> GPIO11

J3 (Shoulder B):
- STEP -> GPIO12
- DIR  -> GPIO13
- EN   -> GPIO14
- PDN_UART -> GPIO15

J4 (Elbow):
- STEP -> GPIO16
- DIR  -> GPIO17
- EN   -> GPIO18
- PDN_UART -> GPIO21


Notes:
- 1k series resistor on each PDN_UART
- 4.7k pull-up to 3.3V on PDN_UART
- EN controlled by Safety Task
- Common GND required


Servos (6V Rail):

Wrist Pitch -> GPIO35
Wrist Yaw   -> GPIO36
Gripper     -> GPIO37

- Powered from 6V buck
- Common ground with MCU
- Separate high-current trace routing


UART from Leader (dedicated hardware UART recommended):

RX (Leader TX) -> GPIO38
TX (Leader RX) -> GPIO39


------------------------------------------------------------
3.4 USB Architecture (Major Update)
------------------------------------------------------------

Integrated USB Hub on Follower PCB

Architecture:
- ESP32-S3 (Native USB)
- ESP32-S3-CAM (Native USB)
- Additional USB-A Port 1 (Camera Expansion)
- Additional USB-A Port 2 (Future Device)

All connected to onboard USB 2.0 Hub IC
Single upstream USB-C connection to laptop


Hub Layout Concept:

Upstream:
- USB-C connector
- ESD protection
- 90Ω differential routing
- Hub upstream port

Downstream Ports:
- ESP32-S3
- ESP32-S3-CAM
- USB-A Port 1
- USB-A Port 2


Design Rules:
- Keep D+ / D− matched length
- 90Ω differential impedance
- Hub powered from 5V rail
- Proper decoupling on hub IC
- Separate analog ground stitching near USB

No data connection between S3 and S3-CAM internally.
Both enumerate independently on laptop.


------------------------------------------------------------
3.5 Power Architecture (Finalized)
------------------------------------------------------------

Input:
- 24V 10A main input


Rail Breakdown:

24V Rail:
- TMC2209 VM
- Direct motor supply

6V Rail (Buck Converter):
- MG996R servos
- Large bulk capacitance near servo headers
- Separate current return routing to reduce noise injection

3.3V Rail (Buck Converter):
- ESP32-S3 (Follower)
- ESP32-S3-CAM
- TMC2209 VIO
- Pull-ups / logic

5V Rail (USB Domain):
- USB hub IC
- USB VBUS to downstream devices
- Generated via buck from 24V or derived from regulated source


Grounding Strategy:
- All grounds common at star point near power entry

Important:
- Keep servo return paths away from MCU ground reference
- USB shield connected via RC / ESD network

4. Embedded Firmware Architecture Update
------------------------------------------------------------

Control Frequency:
- Motor control loop: 200–500 Hz
- VLA policy frequency: 5–10 Hz
- USB telemetry streaming continuous


Updated Communication Stack:

Follower -> Laptop:
- USB-CDC:
  - Joint positions (6 DOF)
  - Timestamp
  - Status
  - Optional telemetry
- USB-UVC:
  - Camera stream (ESP32-S3-CAM)

Leader -> Follower:
- UART:
  - Absolute joint targets
  - Deterministic packet
  - Low latency
  - Independent of laptop


------------------------------------------------------------
5. Mechanical Implications (Integrated)
------------------------------------------------------------

Shoulder Dual NEMA17

Reason:
- Shoulder is highest torque joint
- 500g @ 500mm reach target
- Cycloidal gearbox 25:1
- No bearings → torque margin required

Two motors:
- Share torque load
- Reduce shaft bending
- Increase reliability


------------------------------------------------------------
6. ROS2 Update (Laptop Side)
------------------------------------------------------------

Unchanged philosophy from original but clarified data flow.

Nodes:
- usb_bridge_node
- joint_state_node
- camera_node
- vla_inference_node
- safety_node


Data Alignment Strategy:
- Image received
- Match to nearest joint_state timestamp
- Next joint_state = action label
- Store structured episode

Minimal rosbag schema:
- sensor_msgs/Image
- sensor_msgs/JointState
- custom ActionMsg
- timestamp


------------------------------------------------------------
7. Safety Architecture
------------------------------------------------------------

Critical Safety Rules:
- Motors disabled by default on boot
- Heartbeat required to enable motion
- Heartbeat timeout: ~500 ms
- Driver EN pin controlled by safety task

This prevents runaway motion if the laptop freezes.


------------------------------------------------------------
8. Mechanical Design Constraints (For Mech Team)
------------------------------------------------------------

- Cycloidal gearboxes designed for 3D printing
- Backlash tolerated in software
- Joint hard-stops aligned with StallGuard homing
- Wiring routed internally where possible
- Camera mount must avoid occlusion by wrist


------------------------------------------------------------
9. Cost & Sustainability Goals
------------------------------------------------------------

- Reuse NEMA 17 motors from old 3D printers
- Reuse MG996R servos
- 3D-printable structure
- Target BOM: ~$250 (excluding prints)


------------------------------------------------------------
10. Non-Goals (Important)
------------------------------------------------------------

- On-device AI inference
- High-speed industrial precision
- Closed-loop vision control on ESP32
